개요 — OPM이란 무엇인가

도메인. 이 프로그램은 Park Systems XE 장비의 Sliding Stage(이동 스테이지)가 같은 위치를 반복 측정했을 때 표면 프로파일이 얼마나 일관되게 재현되는지(반복성, Repeatability)를 정량화합니다. 웨이퍼 위 9개 지점을 N회 반복 측정한 1차원 높이 프로파일(.tiff)을 입력으로 받습니다.

OPM의 정의. OPM(Optical Profiler Measurement)은 한 개 프로파일의 최고점−최저점(Peak-to-Valley), 즉 높이 범위(nm)입니다.

OPM(nm) = max(z_nm) − min(z_nm)

src/core/tiff_reader.py:77-79 — ProfileData.opm_nm property. 이는 가공 전(raw) 단일 프로파일의 OPM이며, 실제 분석 지표는 Flatten·Outlier 처리 후 재계산됩니다(§6–7).

4대 핵심 지표 (한 위치 · N repeats 기준)

지표한 줄 의미단위
Rep. Max같은 위치에서 repeat 간 픽셀별 높이 편차의 최대값nm
Rep. 1σ각 픽셀의 repeat 간 표준편차를 RMS로 집계한 값 (반복성 Spec 판정에 사용)nm

4.3 Z축 물리 단위 변환 (DAC → nm)

부호(sign) 보존이 핵심 Z 감도는 부호를 가집니다(Height 채널은 음수). XE Software 규약과 프로파일 방향을 맞추기 위해 부호를 그대로 유지합니다. 절댓값을 취하지 않습니다. tiff_reader.py:157-161

전처리 — Flatten & Outlier

6.1 Order-1 Edge-Only Flatten (분석용 표준)

프로파일은 기울어져 있어 분석 전 수평 보정(Flatten)이 필요합니다. 분석의 모든 지표는 "양끝 1% 픽셀만으로 1차 회귀를 적합하고, 그 회귀선을 전체 구간에서 빼는" 방식을 공통 사용합니다.

edge_pixels = max(1, int(N * edge_percent / 100)) # 8192 → 81px (각 끝) x_fit = concat( x[:edge_pixels], x[-edge_pixels:] ) # 양끝만 z_fit = concat( z[:edge_pixels], z[-edge_pixels:] ) # 수치 안정화를 위해 x 정규화 후 polyfit x_norm = (x - x.mean()) / x.std() coeff = polyfit(x_fit_norm, z_fit, order=1) regression = polyval(coeff, x_norm) z_flat = z - regression # 전체에서 차감

출처 flatten.py:231-270 edge_only_flatten(order=1, edge_percent=1.0)

왜 "양끝만"인가 비유하자면, 자(ruler)의 양쪽 끝만 바닥에 맞추고 가운데는 그대로 두는 방식입니다. 전체 구간으로 fitting하면 가운데의 Bow(휨)가 일부 제거되어 버리지만, 양끝 fitting은 Bow를 보존합니다. 이것이 연구소 기준 Tool과 결과를 일치시키는 핵심 차이입니다 (PMS Q&A 4249).

6.2 Outlier Pixel 제외

같은 위치를 N번 측정했을 때 특정 픽셀에서 1회만 비정상적으로 큰 값이 나올 수 있습니다. 이를 repeat 간 Max−Min 범위가 큰 픽셀을 제외해 걸러냅니다.

stack = [z_flat_r1, …, z_flat_rN] # (repeats, pixels) pixel_range = stack.max(axis=0) − stack.min(axis=0) # (pixels,) # percentile 모드 (value=5 → 상위 5% 제외) threshold = percentile(pixel_range, 100 − value) valid_mask = pixel_range <= threshold # pixels 모드 (value=10 → 범위 상위 10개 제외) valid_mask[ argsort(pixel_range)[-value:] ] = False

출처 analyzer.py:128-149 _exclude_outlier_pixels(). 기본값 mode="none"(제외 없음). 실제 사례: 3_RT/Sample18의 148nm 이상값 1개가 Rep. Max를 6.3배 과대 산출 → outlier 모드로 정상화.

pixel_range = stack.max(axis=0) − stack.min(axis=0) Rep. Max = pixel_range[valid].max()

7.2 Rep. 1σ — 반복성 Spec의 핵심

각 픽셀에서 repeat 간 표준편차(ddof=0)를 구한 뒤, 그 값들을 RMS로 집계합니다. 단순 std(pixel_range)가 아님에 유의.

pixel_stds = stack.std(axis=0, ddof=0) # 픽셀별 repeat std Rep. 1σ = sqrt( mean( pixel_stds[valid]² ) ) # RMS 집계 (repeats ≥ 2)

7.3 OPM Max

각 repeat의 유효 픽셀 Max−Min(=그 repeat의 OPM) 중 최대값.

OPM[r] = stack[r, valid].max() − stack[r, valid].min() OPM Max = max_r( OPM[r] )

7.4 OPM 1σ

모든 repeat × 유효 픽셀 높이값 전체에 대한 zero 기준 RMS. 평균을 다시 빼지 않으므로 "from zero"이며, 이는 반복 변동성이 아니라 평탄화된 프로파일 형상(Bow)의 크기를 나타냅니다.

all_heights = stack[:, valid].ravel() # 전체 repeat × 유효 픽셀 OPM 1σ = sqrt( mean( all_heights² ) )

해상도 정규화 / 리샘플

Range마다 픽셀 해상도가 다르므로(§4.4), 서로 다른 Range를 공정하게 비교하려면 공통 해상도로 맞춥니다. 블록 평균(block-average) 다운샘플을 사용합니다.

res_nm = scan_size_um * 1000 / pixel_count factor = max(1, round(target_res_nm / res_nm)) # block-average 다운샘플 n_new = len(z) // factor z_ds = z[:n_new*factor].reshape(n_new, factor).mean(axis=1)

출처 analyzer.py:449-514 (resample_profile, compute_normalized_opm). 정규화 OPM은 리샘플 후 polynomial_flatten(order=1)로 레벨링하고 Max−Min을 취함.

비교 분석 (Comparator)

두 분석 결과(현재 vs 기준)를 위치별로 비교해 차이(delta)와 변화율을 계산합니다. 다중 레시피/조건 비교에 사용.

  • 위치별 delta: rep_max, rep_1sigma, opm_max 각각의 (현재 − 기준)과 OPM Max 변화율(%).
  • 그룹 요약: POSITION_GROUPS(Center/Side/Edge)별 평균·최대 OPM.
  • 비교 소스: use_best_window=True면 Best-5 위치, 아니면 전체 위치.

Ball Screw / Dishing (옵션)

CMP(화학적 기계 연마) 표면 손상 지표인 Dishing/Erosion을 XE의 CMP Percentile 방식으로 계산하는 부가 모듈입니다.

# 1) 12차 다항식 flatten (전체 구간) flattened = polynomial_flatten(z, order=12) # 2) Base zone (offset 10%, range 10%) → ErosionTop = 50th pct # 3) Interest zone (0~100%) → DishingTop = 99th, DishingBottom = 1st pct Erosion = DishingBottom − ErosionTop # 음수면 base보다 침식 Dishing = DishingTop − DishingBottom

출처 ball_screw_analyzer.py:116-183 compute_dishing(). Spec: AL ≤ 6.0 nm, SUS ≤ 4.5 nm (SPEC_DISHING ball_screw_analyzer.py:34-37), 판정은 repeat 간 MAX Dishing 기준(B-plan).

코드 vs docstring 모듈 docstring(L11)은 "Erosion = ErosionTop − DishingBottom"으로 적혀 있으나 실제 코드(L180)는 DishingBottom − ErosionTop입니다. 본 문서는 실제 코드를 기준으로 합니다.

  5. Best-5 / Spec: §8, §9 (장비 타입 dw/iso 분기).
